Day 2 - July 11, 2026
File: Day2TypeCasting.java
Topic: Type casting - implicit vs explicit casting
Notes: Implicit casting happens automatically (small type -> big type).
       Explicit casting needs manual (type) syntax (big type -> small type),
       and can lose data (e.g. double 99.99 -> int 99, not rounded).
       Used Integer.parseInt() for String to int, and String.valueOf()
       for int to String.
Status: Completed

Day 3 - July 12, 2026
File: Day3ArithmeticOperators.java
Topic: Arithmetic operators - +, -, *, /, %
Notes: Division (/) between two int values drops the decimal part
       (e.g. 10/3 = 3, not 3.33). Must cast one value to double to
       get a decimal result. Java has no ** operator - use Math.pow()
       for exponents instead.
       Math class comes from java.lang, which is auto-imported in every Java
       program - no import statement needed. It contains built-in methods like
       pow(), sqrt(), abs(), max(), min(), round().
Status: Completed

Day 4 - July 17, 2026
File: Day4ComparisonOperators.java
Topic: Comparison operators - ==, !=, >, <, >=, <=
Notes: For numbers, == works normally. But for Strings, == compares memory
       location, not actual text - unreliable. Must use .equals() to
       correctly compare String content in Java.
Status: Completed

Day 5 - July 18, 2026
File: Day5LogicalOperators.java
Topic: Logical operators - &&, ||, !
Notes: Same logic as Python's and/or/not, but Java uses symbols instead
       of words. && needs both true, || needs at least one true,
       ! flips true/false.
Status: Completed

Day 6 - July 19, 2026
File: Day6IfElse.java
Topic: If-else statements - if, else if, else, nested if
Notes: Java uses "else if" (two words) instead of Python's "elif".
       Curly braces {} define each block instead of indentation.
       Nested if works the same concept as Python, just wrapped in {}.
Status: Completed
